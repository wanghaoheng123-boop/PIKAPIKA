# Contributing

## Package boundaries

- Feature code goes in a `Packages/*` module, not the app target. The app
  targets are thin composition roots (`PikaApp.swift` + top-level views).
- A package may depend on `PikaCore` only. If two feature packages share
  something, promote it to `PikaCore`.
- `SharedUI` is allowed to depend on `PikaCore` for value types used in
  accessibility labels — do not leak view logic back into `PikaCore`.

## Swift style

- `SWIFT_STRICT_CONCURRENCY=complete` is on. Fix warnings; do not silence.
- Prefer `actor` over `@MainActor` classes unless the type holds UI state.
- Mark protocols `Sendable` when they cross actor boundaries.
- Public API gets a doc comment; private helpers usually don't.

## Testing

Every package has a `Tests/` target. Targets that don't compile for all
platforms (e.g. `AppKit`-dependent activity monitor) must guard with
`#if os(macOS)`.

Run package tests with:
```bash
cd Packages/<Name> && swift test
```

## Commits

- Conventional-ish. Imperative mood. One logical change per commit.
- Don't commit generated `*.xcodeproj` bundles — they're regenerated by
  `Scripts/generate-xcode.sh`.
